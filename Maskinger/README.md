# Makinger

AIに入力する前のテキストを、ブラウザ内で自動マスキングする単体HTMLツールです。

## できること

- 入力欄に打ち込んだ瞬間に、右側へリアルタイムでマスキング結果を表示
- レイアウトを分割表示（左:入力 / 右上:マスク結果 / 右下:マスキング対応一覧）
- 「ヘルプ」ボタンで、機能概要・主なマスク対象・使い方をモーダル表示
- 通常テキストの機密情報をマスク
  - メールアドレス
  - URL
  - IPアドレス
  - 電話番号
  - SSN 形式 (`123-45-6789`)
  - 郵便番号 (`〒123-4567` / `123-4567`)
  - IBAN 形式
  - UUID
  - クレジットカード番号（Luhnチェックで妥当性確認）
  - 12桁の個人番号形式
  - ローカルパス
  - 秘密情報っぽい値 (`password`, `token`, `api_key` など)
  - PII系キーへの代入値 (`email`, `phone`, `address`, `account_number`, `user_id` など)
  - 秘密鍵ブロック (`-----BEGIN ... PRIVATE KEY-----`)
  - Bearer トークン
  - 認証情報入り接続文字列 (`postgres://user:pass@...` など)
  - 主要API/アクセストークン形式（OpenAI, GitHub, Anthropic, Google, Stripe, Slack, AWSなど）
  - 敬称付き氏名、会社名らしき文字列
- コードらしい入力の場合は追加で自動処理
  - コメントを削除 (`//`, `/* */`, `#`, `--`)
  - SQL文に見える文字列リテラルを `[SQL_n]` に置換
  - クラス名 / 型名（`class`, `interface`, `enum`, `type`）を `[CLASS_n]` に置換
  - 関数名（`function` / `def` / `func` 宣言、`const|let|var` への関数代入、Java系メソッド宣言、クラス/オブジェクトメソッド）を `[FUNC_n]` に置換
  - 定数名（`UPPER_SNAKE_CASE` など）を `[CONST_n]` に置換
  - 固定文字列リテラル（`"..."`, `'...'`, `` `...` ``）を `[STRING_n]` に置換
- マスキング後一覧を縦の箇条書きで表示
  - 形式: `[PLACEHOLDER_n] = 元データ`
- 出力欄/一覧には専用コピー操作を用意（各カード右上）
  - 「コピー」: マスク済みテキスト全体
  - 「マスク後一覧コピー」: マスク後側の一覧
- コピー成功時は、ボタン表示と色が一時的に変化（「コピーしました」表示）

## 使い方

1. `Maskinger.html` をブラウザで開きます。
2. 必要に応じて「ヘルプ」を押して、対象情報や注意事項を確認します。
3. 入力欄にテキストまたはコードを貼り付けます。
4. 右上の出力欄でリアルタイム反映されたマスク結果を確認します。
5. 右下の一覧で「マスク後プレースホルダーと元データ」の対応関係を確認します。
   - 入力欄が空の間は、一覧に補助メッセージは表示されません。
6. 必要に応じて「コピー」「マスク後一覧コピー」を使います。

## 補足

- 変換はブラウザ内で完結し、外部送信は行いません。
- マスキングはルールベースです。文脈依存の固有名詞は取りこぼす場合があるため、送信前の最終確認を推奨します。
