# WEW (Web Editor Web)

`ArtEdit.html` は、ローカルで動作する単一HTMLのWebエディタです。

## このフォルダの使い方

1. `ArtEdit.html` をブラウザで開く
2. 新規作成 / 開く / 保存 / 別名保存などを上部メニューで操作する

## 日本向けリファクタリング内容

- UI文言を日本語化（新規/保存/リネーム/ヘルプ/設定/About）
- 新規HTMLテンプレートに `lang="ja"` を追加
- 設定ダイアログ・ショートカット説明を日本語化
- `document.documentElement.style.colorScheme` をテーマ連動で反映
- 外部CDNのLint読込を削除し、Lintはライブラリ存在時のみ有効化
- メニュー/ファイル種別アイコンを Material Icons（CDN）へ置換
- Googleドキュメント風のUIへリデザイン（ライトグレーキャンバス + 白いツールバー/編集面 + 青アクセント）
- 1ファイル編集向けの固定ツールバーへ再構成（ファイル名・開く・保存・別名保存・Undo/Redo・検索・整形・設定・ヘルプを常設）
- ツールバーの各ボタンをアイコン表示へ統一し、独自ツールチップ（黒背景・白文字）を追加
- 下部ステータスバーを追加（行/列、言語、Lint状態、未保存変更）
- Roboto + Noto Sans JP ベースのタイポグラフィへ変更
- ツールバー/ダイアログの Material Icons をボタン中央に揃えるようスタイル修正
- ダイアログ（`jsPopupBox`）を刷新し、レスポンシブなモーダルレイアウト・統一ボタン・Escape/閉じる導線を改善
- モーダルをSaaSで一般的な3セクション構成（ヘッダー / 本文 / フッター）へ全面刷新し、全モーダルでタイトル`1.2rem`・本文`0.85rem`を統一
- フォームモーダルの項目配置を行単位レイアウトへ再設計し、不要な`<br>`由来の余白や幅崩れを解消
- フォーム値の収集処理をJSON文字列連結からオブジェクト構築へ変更し、貼り付け文字列（`\`を含む入力）で画面が復帰しない不具合を修正
- エディタの初期フォント倍率を `0.9rem` へ調整し、設定から `0.75` 以上で細かく変更可能
- 検索/置換ボタン押下時にメニュー親のフォーカス処理でダイアログが即閉じる不具合を修正
- 検索UI（CodeMirrorダイアログ）を右上コンパクトカード型へ刷新し、検索入力・補助テキスト・配色を再設計
- 検索/置換を単一の常設2段パネル（検索行 + 置換行）へ統合し、置換確認ダイアログを廃止して即時置換（`置換`/`すべて置換`）へ変更
- 検索パネルの文言を日本語化し、移動/置換ボタンをアイコン表示（文字ラベル非表示）へ変更
- 置換ツールバーボタンを削除し、検索ボタンとショートカット（`Ctrl/Cmd+F`, `Ctrl+H`/`Cmd+Alt+F`, `F3`/`Shift+F3`）を一般的なエディタ仕様へ統一
- 設定ダイアログのフォントサイズ入力バリデーション（`step`）を修正し、テーマ変更が保存・即時反映されない不具合を修正
- モーダル幅を固定値から内容別可変（`compact`/`form`/`wide` = 420/520/620px目安）へ変更し、不要な横余白を削減
- `setTheme()` のUI色強制上書きを整理し、エディタテーマ切替とデザイン一貫性を両立
- メニュー表示/非表示ボタンとF10トグルを削除し、上部メニューを常時表示に変更
- 保存動作を分離（`保存` は現在ファイル名で即保存、`別名保存` はファイル名入力後に保存）

## 補足

- このツールはローカルファーストで動作します。
- 外部ネットワーク通信は行いません（Material Icons CDN 読込と、ユーザーが明示的に外部リンクを開く場合を除く）。
