禁止ルール

  • var は禁止です。
  • 文字列は基本的に 'single quote' を使います。"double quote" は警告対象です。
  • 比較演算子は === と !== を使います。
  • == null と != null は例外として許可します。
  • .value === と .value !== の比較では、比較相手にシングルクォートの文字列を使います。

自動整形ルール

  • 制御構文やブロックでは、キーワードの後ろと { の前に半角スペースを 1 つ入れます。
  • セミコロンは基本的に付与します。
  • ただし、関数宣言の末尾には付与しません。
  • インデントは半角スペース 2 個に統一します。
  • 行末の余計な空白は削除し、改行コードは LF に統一します。

このツールは、コードの意味を変える修正ではなく、書式統一と基本的な禁止ルールの検出に絞っています。
